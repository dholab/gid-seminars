# CDC Seminars and Webinars - Programmatic Access Research

Research Date: 2026-01-14

## Executive Summary

This research identifies programmatically accessible feeds for CDC infectious disease seminars, webinars, and educational content. While CDC offers extensive educational programming, **direct programmatic access is limited**. Most webinar series rely on email subscriptions (GovDelivery) and YouTube playlists rather than dedicated RSS or iCal feeds.

### Key Findings:
- **RSS Feeds Available**: Primarily for publications (MMWR, EID journal), travel notices, and some topic-specific content
- **Webinar Series**: Multiple active series but minimal programmatic access beyond YouTube
- **API Access**: Content Syndication API available for multimedia content
- **iCal/Calendar Feeds**: NOT FOUND - No dedicated calendar subscription feeds identified
- **Primary Access Method**: Email subscriptions via GovDelivery

## 3. APIs and Structured Data

### 3.1 CDC Content Syndication API

**Base Endpoint:** `https://tools.cdc.gov/api`

**Documentation:** https://tools.cdc.gov/api/docs/info.aspx

**Response Formats:** JSON, JSONP, XML

**Key Endpoints:**

1. **Media Search**
   - `GET /v2/resources/media` - List all available media
   - `GET /v2/resources/media?q={STRING}` - Full-text search
   - `GET /v2/resources/media?topic={STRING}` - Filter by topic
   - `GET /v2/resources/media/{MEDIA_ID}` - Get specific resource

2. **Topics**
   - `GET /v2/resources/topics` - List available topics

3. **Syndication**
   - `GET /v2/resources/media/{MEDIA_ID}/syndicate` - Get HTML content
   - `GET /v2/resources/media/{MEDIA_ID}/embed` - Get embed code

**Useful Parameters:**
- `mediaTypes` - Filter by content type (HTML, Video, Infographics, Widgets)
- `audience` - Target specific groups (Healthcare Providers, etc.)
- `topicIds` - Filter by subject
- `languageName` / `languageIsoCode` - Language filtering

**Use Case:** Can potentially search for educational content, but **does not appear to have specific webinar/event endpoints**

**Status:** ACTIVE but not optimized for discovering upcoming events

### 3.2 CDC Open Data Portal (data.cdc.gov)

**Platform:** Socrata Open Data API

**Main Portal:** https://data.cdc.gov/

**API Documentation:** https://dev.socrata.com/

**Developer Resources:** https://open.cdc.gov/apis.html

**Content Categories:**

## 9. Recommendations for Programmatic Access

### For Content Aggregation Projects:

**RECOMMENDED APPROACH:**

1. **Use RSS Feeds for Publications:**
   - Subscribe to MMWR and EID feeds for published content
   - Monitor Travel Notices for outbreak updates

2. **YouTube RSS for Recorded Webinars:**
   - Identify CDC YouTube playlist IDs
   - Subscribe to playlist RSS feeds
   - Provides automated updates for new recordings

3. **Email Monitoring for Upcoming Events:**
   - Subscribe to GovDelivery email updates
   - Parse emails programmatically (if needed)
   - COCA, ZOHU, and other series primarily use email

4. **Web Scraping as Backup:**
   - Monitor webinar series pages for updates
   - Parse HTML for event listings
   - Use as fallback when RSS/email unavailable

5. **CDC Content Syndication API:**
   - Query for educational media content
   - Filter by topics relevant to infectious disease
   - Note: Not optimized for event discovery

### Not Recommended (Not Available):

- iCal calendar subscriptions (don't exist)
- Dedicated webinar/event API (not available)
- JSON event feeds (not found)

## 10. Gap Analysis

### What CDC Does Well:

- High-quality RSS feeds for journal publications (MMWR, EID)
- Comprehensive email notification system
- YouTube integration for webinar recordings
- Free continuing education opportunities
- Multiple topic-specific webinar series

### What's Missing:

1. **Unified Event Calendar** - No centralized calendar feed for all CDC educational events
2. **iCal Subscriptions** - No .ics calendar feeds for any webinar series
3. **Webinar API** - No structured API for discovering upcoming webinars
4. **JSON Event Feeds** - No machine-readable event listing feeds
5. **Consistent RSS Implementation** - Some webinar series mention RSS but don't provide URLs
6. **Playlist IDs Documentation** - YouTube playlists exist but IDs not always documented

### Impact:

Organizations wanting to programmatically aggregate CDC webinar information must rely on:
- Email parsing (GovDelivery notifications)
- Web scraping (individual webinar series pages)
- Manual monitoring and updates

This creates barriers for automated aggregation and reduces discoverability of CDC educational content.

## 12. Technical Implementation Notes

### RSS Feed Consumption:

```python
# Example: Fetching MMWR RSS feed
import feedparser

mmwr_feed_url = "https://tools.cdc.gov/api/v2/resources/media/342778.rss"
feed = feedparser.parse(mmwr_feed_url)

for entry in feed.entries:
    print(f"Title: {entry.title}")
    print(f"Link: {entry.link}")
    print(f"Published: {entry.published}")
```

### YouTube RSS Feed Format:

```
https://www.youtube.com/feeds/videos.xml?playlist_id={PLAYLIST_ID}
```

To find playlist ID:
1. Visit CDC YouTube channel
2. Navigate to Playlists
3. Click on desired playlist
4. Extract ID from URL: `youtube.com/playlist?list=PLAYLIST_ID`

### Content Syndication API Example:

```bash
# Search for infectious disease media
curl "https://tools.cdc.gov/api/v2/resources/media?q=infectious+disease&mediaTypes=Video"
```
